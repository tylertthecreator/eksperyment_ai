Zbieranie wyników (CSV) do Google Arkusza — za darmo
====================================================

1) Nowy arkusz Google (np. nazwa „Wyniki eksperymentu”).
2) Rozszerzenia → Apps Script → wklej kod z pliku:
   google-apps-script/append-results.gs
   (zawiera doPost — zapis CSV — oraz doGet — rotacja warunków A→B→C;
   po zmianie kodu: Wdróż → Zarządzaj wdrożeniami → Edytuj → Wdróż nową wersję)
3) Zapisz projekt (Ctrl+S). Upewnij się, że skrypt jest powiązany z tym arkuszem
   (otwarty z menu arkusza).
4) Wdróż → Nowe wdrożenie → koło zębate → „Aplikacja internetowa”:
   - Wykonaj jako: Twoje konto
   - Kto ma dostęp: wszyscy (w tym użytkownicy anonimowi)
5) Skopiuj URL wdrożenia (kończy się często na /exec).

6) Vercel → Twój projekt → Settings → Environment Variables:
   Nazwa: APPS_SCRIPT_URL
   Wartość: wklejony URL z kroku 5
   Środowisko: Production (i Preview, jeśli chcesz testy na branchach)

7) Redeploy projektu (Deployments → … → Redeploy), żeby wczytać zmienną.
   Ten sam URL służy do: POST (wyniki) i GET (przydział warunku) przez /api/submit-results i /api/next-condition.

8) Test online: zakończ eksperyment — w arkuszu powinny pojawić się wiersze.
   Parametr ?nosubmit=1 w adresie wyłącza wysyłkę (tylko pobranie CSV lokalnie).

Lokalnie (npm start / serve): endpoint /api nie działa — to normalne;
pełny test API: npx vercel dev (w katalogu projektu), po zalogowaniu do Vercel.

Uwaga RODO: informuj uczestników o zbieraniu danych i celu; arkusz powinien
być udostępniony tylko Tobie / zespół zgodnie z procedurą badania.
